# 发布备案任务功能 - API接口文档

## 概述
本文档描述了发布备案任务功能的所有API接口。

## 基础信息
- 基础URL: `http://localhost:8080`
- 所有接口支持跨域访问（CORS）

## 接口列表

### 1. 获取课程列表
**接口地址**: `GET /api/courses`

**功能描述**: 查询可选课程列表，支持按年级、学期、状态筛选

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| grade | String | 否 | 年级，如：2021, 2022, 2023 |
| semester | String | 否 | 学期，如：春季、秋季 |
| status | Integer | 否 | 课程状态：0-未指派，1-已指派 |

**响应示例**:
```json
{
  "code": 200,
  "message": "查询成功",
  "data": [
    {
      "id": 1,
      "courseCode": "CS101",
      "courseName": "计算机导论",
      "grade": "2021",
      "semester": "秋季",
      "status": 0,
      "statusDesc": "未指派"
    }
  ]
}
```

---

### 2. 获取教师列表
**接口地址**: `GET /api/users/teachers`

**功能描述**: 查询教师列表，支持按工号、姓名搜索

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| code | String | 否 | 工号（支持模糊搜索） |
| name | String | 否 | 姓名（支持模糊搜索） |

**响应示例**:
```json
{
  "code": 200,
  "message": "查询成功",
  "data": [
    {
      "id": 2,
      "workId": "SWJTU002",
      "realName": "张老师",
      "email": "teacher01@swjtu.edu.cn",
      "phone": "13800138001"
    }
  ]
}
```

---

### 3. 获取审核员列表
**接口地址**: `GET /api/users/assessors`

**功能描述**: 查询可选审核员列表，自动排除指定的教师ID

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | Long | 否 | 要排除的教师ID（当A教师被选为负责人时，自动剔除A教师） |

**响应示例**:
```json
{
  "code": 200,
  "message": "查询成功",
  "data": [
    {
      "id": 4,
      "workId": "SWJTU004",
      "realName": "王评审员",
      "email": "assessor01@swjtu.edu.cn",
      "phone": "13800138003"
    }
  ]
}
```

### 5. 获取任务详情
**接口地址**: `GET /api/tasks/{id}`

**功能描述**: 根据任务ID获取任务详情

**路径参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | Long | 是 | 任务ID |

**响应示例**: 同任务列表中的单个任务对象

---

### 6. 更新任务状态
**接口地址**: `PUT /api/tasks/{id}/status`

**功能描述**: 更新任务状态

**路径参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | Long | 是 | 任务ID |

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| status | String | 是 | 任务状态：PENDING-待处理，IN_PROGRESS-进行中，COMPLETED-已完成，REJECTED-已拒绝 |

**响应示例**:
```json
{
  "code": 200,
  "message": "更新成功",
  "data": null
}
```

---

### 7. 批量发布任务
**接口地址**: `POST /api/tasks/batch`

**功能描述**: 批量创建任务，完成"课程-教师-审核员-时间"的绑定

**请求体**:
```json
{
  "tasks": [
    {
      "courseId": 1,
      "teacherId": 2,
      "assessorId": 4,
      "deadline": "2024-12-31T23:59:59"
    },
    {
      "courseId": 2,
      "teacherId": 3,
      "assessorId": 4,
      "deadline": "2024-12-31T23:59:59"
    }
  ]
}
```

**请求体字段说明**:
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| tasks | Array | 是 | 任务列表 |
| tasks[].courseId | Long | 是 | 课程ID |
| tasks[].teacherId | Long | 是 | 教师ID（负责人） |
| tasks[].assessorId | Long | 是 | 审核员ID |
| tasks[].deadline | String | 是 | 截止日期（ISO 8601格式） |

**业务规则**:
1. 验证课程、教师、审核员是否存在
2. 审核员不能是教师本人
3. 课程不能已被指派（不能有进行中或待处理的任务）
4. 创建任务后，自动更新课程状态为"已指派"

**响应示例**:
```json
{
  "code": 200,
  "message": "批量创建任务成功",
  "data": null
}
```

**错误响应示例**:
```json
{
  "code": 500,
  "message": "审核员不能是教师本人",
  "data": null
}
```

## 任务状态说明

| 状态值 | 状态描述 | 说明 |
|--------|----------|------|
| PENDING | 待处理 | 任务已创建，等待处理 |
| IN_PROGRESS | 进行中 | 任务正在处理中 |
| COMPLETED | 已完成 | 任务已完成 |
| REJECTED | 已拒绝 | 任务已被拒绝 |

---

## 数据表说明

### 课程表 (course)
- 支持按年级、学期筛选
- 状态字段：0-未指派，1-已指派
- 已指派的课程在列表中显示"已指派"状态，防止重复指派

### 任务表 (task)
- 关联课程、教师、审核员
- 包含截止日期和任务状态
- 创建任务时自动更新课程状态

### 用户表 (user)
- 角色为"USER"的用户可作为教师或审核员
- 支持按工号、姓名搜索

## 前端临时任务表

前端可以：
1. 勾选多门课程
2. 为每门课程填入教师、审核员、截止日期等信息
3. 将信息存入前端临时任务表
4. 批量提交，一次性调用批量接口入库

---

## 注意事项

1. 所有日期时间使用ISO 8601格式（如：2024-12-31T23:59:59）
2. 审核员排除逻辑：当A教师被选为某课程的任务负责人时，在"指派审核员"列表里，自动剔除A教师
3. 课程不能重复指派：如果课程已有进行中或待处理的任务，不能再次指派
4. 批量创建任务时，如果其中任何一个任务创建失败，整个事务会回滚
